What is an interim release?

Interim releases are provided between full releases so that important bug fixes and new features can be made available to non-developers earlier than they might otherwise be. Both full and interim releases of of XSB undergoes rigorous testing, and are fully documented.

An interim release can differ from a full release in one of the following ways.

  • The interim release may not have been fully tested on all platforms, or binaries for the release may not have been created (some features are difficult to test under Windows, for instance).
  • The interim release may contain important bug fixes or enhancements, but certain new features may be only partially implemented
  • The interim release may contain important changes, but may have, say, a package that is temporarily broken by a chance.
Another way of looking at it is that each full release is intended to be "at least as good" as the last in all areas, and to contain significant enhancements over the last release. An interim release contains significant enhancements over the previous release and is suitable for most users, but it might not be suitable a small number of users. Release notes for an interim release indicates not only the changes made, but the reason(s) why the release is not full.
